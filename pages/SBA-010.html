Description

Essentially, our aim is to have a usable media player, preferrably, something like Youtube. Which, as it happens, internet helps a lot. We basically stumbled upon this video by Web Dev Simplified that describe the exact steps to recreate Youtube-like media player in html, css, and javascript. He also kindly provides its source code for us to play with.

The player is fully featured. It is able to play and stop with simple keyboard strokes or on the buttons, toggles to fullscreen, theatre mode, miniplayer, turning on caption, change playback speed, volume, and features a fully functioning progress bar with thumbnails. However, we don’t need all of them.

I was hoping that I could remove the thumbnails and preview images because I don’t know how to implement them properly for multiple pages in the same folder. Then for some weird reasons theatre mode doesn’t work pretty well when I use it with pandoc and default styling, so I want to remove that too. Caption is not necessary, because we don’t generally use them. The same for playback speed, I’m not sure if we should keep that.

More importantly, we want to easily add them to a page in pandoc, and not load the javascript and css all the time whenever it is not used. Therefore, in doing so, we have to have a special set of css and javascript for them, that we would only call when we need them. Basically we would need something like the following at the beginning of our code piece:

  <link rel="stylesheet" href="../share/video.css">
  <script src="../share/video.js" defer></script>

Full disclaimer though, I am not a coder, so this is definitely not a comprehensive guide of how to do things generally. Consult the video by Web Dev Simplified youtube channel for full guidance and explanation instead. I’m merely following the video I mentioned to get a gist of what I should do to get the effect I want.

The video.css and video.js file follows closely on Web Dev Simplified’s script, and to achieve what we want (reduce functions), the most obvious solution is to just edit those two files. However, I decided to just create a new set of files: video-simple.css and video-simple.js, so we can always use the full version (when we need to) just by changing the reference.

Removing Thumbnail Images and Preview Images

The first thing I want to take down are thumbnail images and preview images. I start out by isolating style definitions specific for them. It took some try and error, even by inspecting page elements on browser (Ctrl + i on your browser for those who want to try it), until I get what I want. Basically I commented out the styling rules (file video-simple.css) for:

.timeline .preview-img{...}
.thumbnail-img {...}

Then I also commented out all mentions of class .thumbnail-img and class .preview-img from styling rules.

Then we have to take care of the javascript that still rely on those classes, therefore I commented the following lines in video-simple.js:

// const previewImg = document.querySelector(".preview-img")
// const thumbnailImg = document.querySelector(".thumbnail-img")

[...]
